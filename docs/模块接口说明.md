# 三、模块拆分 & 功能设计（核心编码框架）
按功能拆分为 **5 大独立模块**，解耦开发，可单人/多人分工编写

## 3.1 模型模块 `model_module.py` + `llama_engine.py`（双引擎架构）
### 功能职责
1. 自动选择推理引擎：CUDA → PyTorch + bitsandbytes，CPU/集显 → llama.cpp + GGUF
2. Qwen-1.8B 模型加载（Safetensors / GGUF 双格式）
3. 模型离线量化 / 加载量化后模型（INT4/INT8 via bitsandbytes，Q4_K_M via GGUF）
4. 开启自动算子融合 `torch.compile`（仅 PyTorch FP16 + CUDA）
5. 模型层级拆分（按配置分配给主/从节点，仅 PyTorch 引擎支持分布式）
6. 统一对话补全接口 `chat()` / `chat_stream()`

### 对外接口
- `ModelManager.select_engine(profile)`：根据硬件自动选择推理引擎
- `load_model(model_path, quant_type, profile)`：加载模型，自动路由到对应引擎
- `chat(messages, max_tokens, temperature, top_p)`：对话补全（双引擎统一接口）
- `chat_stream(messages, ...)`：流式对话补全
- `split_model(layer_config)`：根据配置拆分模型层（仅 PyTorch）
- `model_forward(input_ids)`：单步前向推理（仅 PyTorch 分布式）

### 配置项
- 推理引擎：`INFERENCE_ENGINE = "auto" | "pytorch" | "llama_cpp"`
- 量化精度 (PyTorch)：`["fp16", "int8", "int4"]`
- GGUF 量化 (llama.cpp)：`["Q4_K_M", "Q5_K_M", "Q8_0", ...]`
- 分层配置：各节点负责的起止层编号

## 3.2 分页KV缓存模块 `paged_kv_cache.py`
### 功能职责
1. 内存页管理、页表映射
2. KV 动态分配、追加写入
3. 会话结束页面回收、清空缓存
4. 适配 Prefill / Decode 双阶段

### 对外接口
- `__init__(page_size)`：初始化页大小、空闲页池、页表
- `append_kv(new_k, new_v)`：追加新Token的KV
- `get_all_kv()`：读取全部历史KV
- `clear()`：清空当前会话缓存，回收所有页面

## 3.3 网络通信模块 `tcp_comm.py`
### 功能职责
1. 主节点（服务端）监听、从节点（客户端）连接
2. 长连接维持、心跳检测、断线重连
3. 封包/解包（长度头 + 数据体）
4. 张量序列化 / 反序列化
5. 控制指令、特征张量收发

### 对外接口
- `start_server(ip, port)`：启动TCP服务端（主节点）
- `connect_client(server_ip, port)`：客户端连接服务端（从节点）
- `send_data(data)`：通用发送接口（自动封包）
- `recv_data()`：通用接收接口（自动解包、还原张量/字符串）
- `heartbeat_loop()`：心跳线程

## 3.4 调度控制模块 `scheduler.py`
### 功能职责
1. 节点状态管理（空闲/忙碌/离线）
2. 推理任务分发、流程启停
3. 异常捕获、错误上报
4. 流水线数据流调度控制

### 对外接口
- `init_nodes()`：初始化所有从节点状态
- `start_infer_task(prompt)`：启动一轮完整推理任务
- `stop_infer_task()`：强制停止推理、重置流水线

## 3.5 前端可视化模块 `frontend/`（React + Vite）
### 功能职责
1. 对话交互窗口（输入提问、展示回答、多轮对话管理）
2. 优化开关：一键开启/关闭 量化、算子融合、分页KV、分布式
3. 实时性能监控：显存占用、CPU使用率、推理时延、Token生成速度
4. 设备面板：GPU/CPU 资源使用率、节点拓扑状态
5. 设置面板：模型参数、推理参数、云端同步配置

### 前端组件
- `App.jsx`：主布局 & 全局状态管理
- `ChatPanel.jsx`：对话交互（多会话、预设问题、单轮删除）
- `DevicePanel.jsx`：设备画像展示（CPU/GPU/内存/磁盘）
- `SettingsModal.jsx`：系统设置 & 优化开关
- `SessionList.jsx`：多会话列表管理

### 技术栈
- React 18 + Vite 5
- FastAPI 后端（开发模式 Vite proxy → 8000，生产模式 FastAPI 直接挂载静态文件）
