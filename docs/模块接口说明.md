# 主要模块与接口说明

> 状态：现行
> 更新日期：2026-07-14
> 注意：本文只列稳定职责和主要入口，不承诺完整 Python API；准确签名以源码和测试为准。

## 1. 模型管理

文件：`src/model_module.py`、`src/llama_engine.py`、`src/model_config.py`

`ModelManager` 统一管理 PyTorch 与 llama.cpp 引擎：

- `select_engine(profile)`：结合显式配置、CUDA 和设备画像选择引擎。
- `load_model(...)` / `unload_model()` / `switch_model(...)`：加载、卸载和切换注册模型。
- `load_layer_range(...)` / `ensure_layer_range(...)`：加载或确认 PyTorch 连续层段及 Embedding/LM Head 权重。
- `ensure_full_model(...)`：在回退完整推理前确认完整模型已加载。
- `forward_layers(...)`：执行 PyTorch 层段前向。
- `chat(...)` / `chat_stream(...)`：完整模型对话入口。

模型注册信息由 `model_config.py` 管理。模型文件存在不等于可以进入分布式流水线，还需模型身份同步、能力检查和层配置 ready ACK。

## 2. TCP 通信

文件：`src/tcp_comm.py`

- `TCPServer.start(...)`：主节点监听并处理注册、认证、心跳和业务消息。
- `TCPServer.send_to_client(...)` / `broadcast(...)`：向已注册节点发送消息。
- `TCPClient.connect(...)` / `disconnect()`：从节点建立或关闭连接，并负责重连状态。
- `TCPClient.send_data(...)` / `recv_data(...)`：发送和接收协议消息。
- `serialize_tensor_fast(...)` / `deserialize_tensor_fast(...)`：层流水线张量序列化。
- `MessageType`：注册、配置、推理转发、层前向、结果、取消和心跳等消息类型。

TCP 连接成功、节点注册成功、模型同步成功和流水线 ready 是四个不同状态，UI 和调度器不能互相替代。

## 3. 调度与层编排

文件：`src/scheduler.py`、`src/graph_orchestrator.py`

`Scheduler` 负责节点生命周期、主从连接、模型同步、层配置、请求队列、流水线执行、取消、指标和回退。主要入口包括：

- `start(...)` / `stop()`：启动或停止调度器。
- `init_nodes()`：初始化节点状态。
- `compute_layer_assignment(...)`：生成满足能力和层覆盖约束的分配结果。
- `run_pipeline_safe(...)`：带就绪检查和回退边界的 PyTorch 流水线入口。
- `start_infer_task(...)` / `stop_infer_task()`：任务生命周期控制。

`PipelineQueue` 提供 FIFO/MLFQ 请求排队、老化、取消和结果管理。`GraphOrchestrator.orchestrate()` 只为 PyTorch 层流水线生成候选路径与层分配，不负责完整 Worker、任务链或张量并行的内部调度。

## 4. KV Cache

文件：`src/paged_kv_cache.py`、`src/model_module.py`、`src/scheduler.py`

`PagedKVCache` 提供固定页、追加、读取和清理能力。分布式 PyTorch 前向实际使用的模型 `past_key_values` 按任务和层段由调度器持有，任务完成或中止时清理。两者不能在文档中混写成同一个实现状态。

## 5. API 与客户端

文件：`src/api_server.py`、`frontend/`、`android/`

- FastAPI 暴露聊天、流式聊天、模型、节点、队列、集群状态和 bootstrap 等接口。
- React PC 前端展示聊天、模型、节点、层分配和执行指标。
- Android Full 支持本地 GGUF 与远程 PC 模式；Lite 保持轻量远程入口。
- Android 当前不注册为 PyTorch `pipeline_worker`。完整 Worker 与任务链仍属于规划能力。

跨模块的目标架构和未来 Provider 接口见[混合分布式推理体系规划](混合分布式推理体系规划.md)。
