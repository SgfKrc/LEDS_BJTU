图转树（最大带宽生成树）算法总结文档

> 状态：现行（仅适用于 PyTorch 层流水线候选路径编排）
> 更新日期：2026-07-14
> 本算法不负责完整推理 Worker、任务链或张量并行调度。文中的全连接拓扑、带宽和显存模型是编排近似，不代表所有节点均实际两两直连，也不保证得到全局最优执行计划。多执行体系见[混合分布式推理体系规划](混合分布式推理体系规划.md)。
 
一、算法应用背景
 
本文面向异构边缘设备上的 PyTorch 层流水线编排。设备可以是台式机、工作站、服务器或笔记本，节点算力、内存和网络状态可能差异很大。调度器以可观测或估算链路构造通信图，再通过启发式方法缩减候选路径搜索空间。
 
为此提出图转树方案：将边缘通信全图转换为最大带宽生成树（最大瓶颈生成树），对拓扑进行降维，把原图高复杂度路径搜索问题转化为多项式时间的树上搜索问题，在保留高带宽链路的同时大幅缩减搜索空间，实现毫秒级节点编排。
 
二、核心概念定义
 
1. 原始拓扑图
 
将边缘集群抽象为无向加权连通图 G=(V, E, w)
 
- V：顶点集合，对应所有边缘推理节点；每个节点附带可用显存、算力属性；
- E：边集合，对应节点间物理通信链路；
- 边权 w：链路通信带宽（核心排序依据），同时附带链路延迟属性。
 
2. 最大带宽生成树
 
也等价于最大瓶颈生成树，满足两大特性：
 
1. 包含原图所有节点，无环路、连通；
2. 优先保留原图中高带宽链路，剔除低带宽劣质链路；
3. 树上任意两点间唯一路径的最小带宽，是原图所有两点路径中带宽的最大值，契合流水线并行对链路带宽的强依赖。
 
3. 算法适用前提
 
边缘推理采用流水线并行（层式拆分部署在不同节点），层间通信量小、对链路带宽敏感，是本算法的主要落地场景。
 
三、算法整体流程
 
整体分为三步：构建原始通信图 → 生成最大带宽生成树 → 树上受限路径搜索，完成节点编排。
 
步骤1：构建全局异构通信图
 
1. 采集所有边缘节点信息：唯一ID、可用显存、峰值算力；
2. 探测节点间两两链路的实时带宽、通信延迟，构建无向边；
3. 形成完整全连接拓扑图，作为算法输入。
 
步骤2：基于Kruskal算法构建最大带宽生成树
 
1. 算法思想
 
沿用经典Kruskal贪心框架，排序规则反转：不再选取最小边，而是按带宽从大到小遍历所有链路，结合并查集（Union-Find）判断连通性，依次选边构建生成树。
 
- 目标：优先保留高带宽链路，保证树结构整体通信能力最优；
- 数据结构：并查集，用于快速判断两节点是否连通、合并连通分量，避免成环。
 
2. 伪代码
 
plaintext
  
函数 MAXBANDWIDTHSPANNINGTREE(节点集合V, 边集合E)
    初始化生成树边集合 T = 空
    初始化并查集 U_F(V)
    // 所有边按带宽降序排序
    E_sorted = 按带宽从大到小排序(E)
    for 每条边 edge(u, v, 带宽) in E_sorted:
        if U_F.Find(u) ≠ U_F.Find(v):
            将 edge 加入生成树 T
            U_F.Union(u, v)  // 合并两个连通分量
    return T
 
 
3. 时间复杂度
 
- 边排序：O(|E|\log|E|)（主导复杂度）；
- 并查集查找/合并：近似常数级（路径压缩+按秩优化）；
整体复杂度：O(|E|\log|E|)。实际耗时取决于实现、节点数和画像采集成本，不能仅凭复杂度承诺“百节点亚毫秒”。
 
步骤3：树上受限路径搜索（DFS深度优先搜索）
 
生成树后，在树形拓扑中搜索满足显存约束 + 端到端延迟最小的流水线路径，完成最终节点编排。
 
1. 约束与优化目标
 
1. 硬性约束：路径上所有节点显存总和 ≥ 大模型整体显存需求；
2. 优化目标：最小化整条流水线的「节点计算总延迟 + 节点间通信总延迟」。
 
2. 搜索规则
 
1. 以树中每个节点依次作为起点，执行DFS遍历；
2. 遍历过程动态累加：当前路径节点总显存、累计计算延迟、累计通信延迟；
3. 一旦总显存满足模型要求，记录当前路径与总延迟，和历史最优对比并更新；
4. 引入剪枝策略提升效率：
- 若当前累计延迟已大于已知最优延迟，直接回溯，不再遍历该分支；
- 若剩余节点显存不足以补足模型需求，提前截断分支。
5. 遍历完成后，输出显存达标、总延迟最小的节点路径，作为流水线初始部署方案。
 
3. 伪代码
 
plaintext
  
函数 FINDPATHWITHMEMREQUIREMENT(生成树T, 模型显存需求M_req)
    最优路径 P_best = 空, 最小总延迟 L_min = 无穷大
    for 每个根节点 r ∈ T的顶点:
        已访问集合V_vis = 空, 当前路径P = 空
        当前总延迟L_cur = 0, 当前总显存M_sum = 0
        // 深度优先递归搜索
        DFS(r, V_vis, P, L_cur, M_sum, M_req, T, P_best, L_min)
    return P_best

// 递归DFS函数
函数 DFS(v, V_vis, P, L_cur, M_sum, M_req, T, P_best, L_min)
    标记v为已访问, 将v加入当前路径P
    M_sum += 节点v可用显存
    L_cur += 节点v计算延迟
    // 满足显存约束，更新最优解
    if M_sum ≥ M_req 且 L_cur < L_min:
        更新 L_min = L_cur
        更新 P_best = P
    // 遍历当前节点所有树邻接节点
    for 每个邻接节点 u ∈ T.neighbors(v):
        if u 未被访问:
            链路延迟 L_edge = 节点v→u通信延迟
            DFS(u, V_vis, P, L_cur+L_edge, M_sum, M_req, T, P_best, L_min)
    // 回溯
    M_sum -= 节点v可用显存
    移除v从P、V_vis
 
 
4. 时间复杂度
 
树形结构下节点与边数量线性，以全节点为起点遍历DFS，整体复杂度 O(|V|^2)，属于多项式复杂度。
 
四、后处理：节点层均衡分配（工程优化）
 
DFS得到最优节点路径后，增加一步工程后处理：
 
1. 初始路径为“贪心塞满”层分配方式，易造成单节点负载不均；
2. 按各节点可用显存占比，重新划分模型连续层区间；
3. 若单节点显存不足，做区间回退调整，在不改变最优节点路径的前提下，均衡各节点计算负载。
 
说明：该步骤不改变图转树+DFS的核心算法逻辑，仅做负载优化。
 
五、算法优势
 
1. 复杂度大幅降低
把全图候选路径问题约束到生成树上，以多项式时间启发式换取更小搜索空间，适配节点动态上下线场景；代价是可能舍弃全图中的更优组合。
2. 链路质量优选
最大带宽生成树天然保留高带宽链路，规避边缘弱网瓶颈，从拓扑层面保障流水线吞吐。
3. 搜索空间精简
树形结构无环路、两点路径唯一，相比全连接图，路径搜索范围极大缩小。
4. 适配边缘特性
兼顾显存硬性约束与延迟优化，完全匹配边缘异构节点、带宽受限的部署环境。
 
六、算法局限性与适配场景
 
1. 适配场景
 
- 主流大模型流水线并行边缘推理集群；
- 节点数量中等（几十至上百节点）、拓扑动态变化的边缘网络；
- 优先保障链路带宽、对端到端延迟敏感的推理服务。
 
2. 局限性
 
1. 仅以带宽作为树构建第一优先级，若边缘网络存在延迟极高但带宽大的链路，可能引入额外延迟（可在路径搜索阶段弥补）；
2. 面向单一流水线编排，多流水线并发场景需结合后续扩容机制协同优化；
3. 极端稀疏拓扑下，树结构会强制连通，可能被迫选用少量低带宽链路。
 
七、整体算法总结
 
图转树（最大带宽生成树）+ 树上DFS路径搜索 是一套面向边缘分布式大模型推理的轻量化节点编排方案。
核心思路：拓扑降维 + 贪心选路 + 约束搜索，先用Kruskal算法将复杂全图转为高带宽生成树，消除NP难问题；再通过带显存、延迟约束的深度优先搜索，快速得到最优流水线节点路径。
 
该算法是当前 PyTorch 层流水线的候选路径编排方法。它不替代所有场景下的 MILP，也不是完整 Worker、任务链、张量并行或 GGUF Stage 的通用调度器；其收益必须通过真实拓扑和基线对照验证。
