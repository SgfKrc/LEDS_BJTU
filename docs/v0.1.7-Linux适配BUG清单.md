> **审查范围**：Linux .deb 打包系统（11个新文件） + 跨平台适配源码（5个修改文件）
> **审查日期**：2026-07-09
> **版本**：v0.1.7

## 严重等级定义

| 等级 | 含义 | 触发条件 |
|------|------|----------|
| **P0** | 严重 | 核心功能完全失效、代码死路径、运行时崩溃 |
| **P1** | 高 | 功能部分失效、用户体验显著受损、安全风险 |

## P0 — 严重（4 个）

### ✅ BUG-P0-01 | tcp_comm.py:254 | Linux `/sys/class/net` 网络检测 = 死代码

- **根因**：`hasattr(socket, '_LOCALHOST')` 始终返回 `False`（Python socket 模块无此属性）
- **影响**：本次适配新增的 Linux 网络类型检测（`/sys/class/net`）完全不可达，Linux 上永远无法通过该路径检测 WiFi/以太网
- **修复**：移除 `hasattr(socket, '_LOCALHOST')` 条件，仅保留 `detected == "unknown"`
- **文件**：[src/tcp_comm.py](../src/tcp_comm.py#L254)

### ✅ BUG-P0-02 | api_server.py:178 | SIGTERM 处理器被 uvicorn 覆盖

- **根因**：模块导入时注册 `signal.signal(SIGTERM, ...)`，但 `uvicorn.run()` 后续覆盖了该处理器
- **影响**：systemd `systemctl stop` 发送 SIGTERM 后自定义优雅关闭逻辑从不执行
- **修复**：方案 A（推荐）— 在 launcher.py 的 uvicorn.run 中传入 `timeout_graceful_shutdown=10`，主进程自行注册 SIGTERM；方案 B — 使用 FastAPI lifespan 事件
- **文件**：[src/api_server.py](../src/api_server.py#L171-L178)

### ✅ BUG-P0-03 | prerm:7-11 | root 权限下 `systemctl --user` 无法管理用户服务

- **根因**：prerm 由 dpkg 以 root 执行，`systemctl --user` 操作 root 的 systemd 用户实例，而非实际桌面用户的
- **影响**：卸载时服务不会停止，成为孤儿进程
- **修复**：将 systemd 服务改为系统级服务（安装到 `/lib/systemd/system/`），prerm 中用 `systemctl stop`（不加 `--user`）
- **文件**：[packaging/linux/prerm](../packaging/linux/prerm#L7-L12)、[packaging/linux/qlh-edge-inference.service](../packaging/linux/qlh-edge-inference.service#L15)、[packaging/linux/build-deb.sh](../packaging/linux/build-deb.sh#L65)

### ✅ BUG-P0-04 | build-deb.sh:92 | 预构建 venv 不可移植

- **根因**：venv 在构建机上创建后打包进 .deb，但 venv 的 `bin/python3` 是绝对路径符号链接，换一台机器 Python 路径不同则符号链接断裂
- **影响**：.deb 安装到目标机器后应用无法启动（`/opt/qlh-edge-inference/venv/bin/python3` → 符号链接目标不存在）
- **修复**：在 `postinst` 中创建 venv 和安装 pip 依赖（安装时在目标系统上执行），构建时仅将 requirements 文件打包进 .deb
- **文件**：[packaging/linux/build-deb.sh](../packaging/linux/build-deb.sh#L90-L104)、[packaging/linux/postinst](../packaging/linux/postinst#L5-L9)

## P1 — 高优先级（7 个）

### ⬜ BUG-P1-05 | control-cuda:26 | "DLL" 应为 ".so"

- **根因**：从 Windows 文档复制粘贴的残余错误。Linux 共享库扩展名是 `.so`
- **影响**：包描述出现 Windows 术语，专业度受损
- **修复**：`s/CUDA 12.x DLL/CUDA 12.x 共享库 (.so)/`
- **文件**：[packaging/linux/control-cuda](../packaging/linux/control-cuda#L26)

### ⬜ BUG-P1-06 | qlh-edge-inference.desktop:7 + build-deb.sh:83 | 图标名不匹配

- **根因**：desktop 文件写 `Icon=qlh-edge-inference`，但 build-deb.sh 将图标复制为 `qlh.png`。系统在 hicolor 主题中查找 `qlh-edge-inference.png`，找不到
- **影响**：Linux 桌面上应用图标不显示
- **修复**：方案 A — `Icon=qlh`（匹配 `qlh.png`）；方案 B — build-deb.sh 中复制为 `qlh-edge-inference.png`
- **文件**：[packaging/linux/qlh-edge-inference.desktop](../packaging/linux/qlh-edge-inference.desktop#L7)、[packaging/linux/build-deb.sh](../packaging/linux/build-deb.sh#L83)

### ⬜ BUG-P1-07 | launcher.py:234 | "yesno" 映射为三按钮 MessageBox

- **根因**：`_MB_YESNO = 0x00000004` 未定义，"yesno" 被错误映射为 `_MB_YESNOCANCEL`（三按钮），多了"取消"
- **影响**：Windows 上 `_prompt_tailscale_setup()` 的 "yesno" 对话框出现多余的"取消"按钮
- **修复**：添加 `_MB_YESNO = 0x00000004` 常量，将 "yesno" 映射改为 `_MB_YESNO`
- **文件**：[packaging/launcher.py](../packaging/launcher.py#L120-L123)、[packaging/launcher.py](../packaging/launcher.py#L234)

### ⬜ BUG-P1-08 | launcher.py:191-195 | Linux zenity "取消" 被映射为 "是"

- **根因**：zenity `--extra-button=取消` 点击后退出码为 0（与"是"相同），当前代码不检查 stderr 中的按钮标签
- **影响**：Linux 上用户在 Tailscale 引导对话框点"取消" → 实际执行为"是" → 无法退出
- **修复**：检查 `rc.stderr` 中是否包含按钮标签"取消"，若是则返回 `_IDCANCEL`
- **文件**：[packaging/launcher.py](../packaging/launcher.py#L186-L195)

### ⬜ BUG-P1-09 | launcher.sh:17 | `activate_this.py` 已废弃

- **根因**：`activate_this.py` 在 Python 3.8+ 标记 deprecated，Python 3.13 彻底移除
- **影响**：使用 Python ≥3.13 的 Ubuntu 系统上 venv 激活静默失败 → ImportError
- **修复**：改用 `site.addsitedir()` 手动添加 site-packages 到 sys.path
- **文件**：[packaging/linux/launcher.sh](../packaging/linux/launcher.sh#L14-L19)

### ⬜ BUG-P1-10 | postinst:10 | `chmod 777` 安全风险

- **根因**：`chmod 777` 使模型和日志目录对系统所有用户可写
- **影响**：任意本地用户可删除/替换模型文件（模型投毒）、写入任意日志
- **修复**：改为 `chmod 755`（或 `775` + 合适属组）
- **文件**：[packaging/linux/postinst](../packaging/linux/postinst#L10)

### ⬜ BUG-P1-11 | device_profiler.py:725/771 | nvidia-smi 与 lspci 可能重复检测

- **根因**：nvidia-smi 名称 `"NVIDIA GeForce RTX 3060"` ≠ lspci 名称 `"NVIDIA Corporation GA106 [GeForce RTX 3060]"`，字符串全名匹配无法去重
- **影响**：同一 GPU 重复出现（一个 cuda_available=True，一个 cuda_available=False）
- **修复**：提取 lspci 名称中的规范 GPU 型号（方括号内容）进行去重，或检查是否已有同名 NVIDIA GPU
- **文件**：[src/device_profiler.py](../src/device_profiler.py#L761-L772)

## P2 — 中优先级（9 个）

### ⬜ BUG-P2-12 | launcher.py:723 | `_kill_port_8000()` 中 sock 重复 bind 不可靠

- **根因**：循环中 sock.bind() 在首次成功后再次调用时行为不确定（Windows 抛异常，Linux SO_REUSEADDR 允许）
- **影响**：端口占用检测在跨平台场景下行为不一致
- **修复**：改用 `socket.connect(("127.0.0.1", 8000))` 检测端口占用
- **文件**：[packaging/launcher.py](../packaging/launcher.py#L719-L731)

### ⬜ BUG-P2-13 | launcher.py:134 | `str | None` 语法需 Python 3.10+

- **根因**：PEP 604 联合类型语法（`X | Y`）需要 Python 3.10+
- **影响**：Python 3.8/3.9 上 SyntaxError（但 Ubuntu 22.04 默认 Python 3.10，影响有限）
- **修复**：文件顶部添加 `from __future__ import annotations`
- **文件**：[packaging/launcher.py](../packaging/launcher.py#L1-L5)

### ⬜ BUG-P2-14 | model_downloader.py:502 | ModelScope 下载创建了父目录而非目标目录

- **根因**：`os.makedirs(os.path.dirname(safetensors_dir), exist_ok=True)` 创建父目录，与 HuggingFace 路径的 `os.makedirs(safetensors_dir, exist_ok=True)` 不一致
- **影响**：`snapshot_download` 可能需要自行创建目标目录，不可靠
- **修复**：`s/os.path.dirname(safetensors_dir)/safetensors_dir/`
- **文件**：[src/model_downloader.py](../src/model_downloader.py#L502)

### ⬜ BUG-P2-15 | model_downloader.py:429 | `os.rename()` 无法跨文件系统

- **根因**：`os.rename(src, dst)` 在 src 和 dst 位于不同文件系统时失败
- **影响**：下载缓存（如 `/tmp/`）与 `models/` 不在同一挂载点时移动文件失败
- **修复**：改用 `shutil.move(src, dst)`
- **文件**：[src/model_downloader.py](../src/model_downloader.py#L429)

### ⬜ BUG-P2-16 | device_profiler.py:679/783 | AMD Radeon PRO 被误分类为集显

- **根因**：`"radeon" in name and "rx" not in name → is_igpu = True` 过于宽泛。Radeon PRO 系列（如 W5700）不含 "rx"
- **影响**：AMD Radeon PRO 工作站显卡被错误识别为集成显卡
- **修复**：在 dgpu_kw 中添加 `"radeon pro"` 关键词，先判断独显再判断集显
- **文件**：[src/device_profiler.py](../src/device_profiler.py#L679)、[src/device_profiler.py](../src/device_profiler.py#L783)

### ⬜ BUG-P2-17 | control-cpu:8 + control-cuda:8 | tailscale 应为 Recommends 而非 Depends

- **根因**：Tailscale 不在 Ubuntu 官方仓库中，需要手动添加 APT 源
- **影响**：`dpkg -i` 安装依赖检查失败，需要额外 `apt-get install -f`
- **修复**：将 `tailscale` 从 `Depends` 移到 `Recommends`
- **文件**：[packaging/linux/control-cpu](../packaging/linux/control-cpu#L8)、[packaging/linux/control-cuda](../packaging/linux/control-cuda#L8)

### ⬜ BUG-P2-18 | prerm/postrm | 未处理 dpkg 动作参数 `$1`

- **根因**：prerm/postrm 忽略 `$1`（remove/purge/upgrade），所有场景行为相同
- **影响**：upgrade 场景也删除符号链接（虽 postinst 会重新创建，但不优雅）；purge 不清理数据
- **修复**：添加 `case "$1" in remove|purge|upgrade) ...` 分支
- **文件**：[packaging/linux/prerm](../packaging/linux/prerm)、[packaging/linux/postrm](../packaging/linux/postrm)

### ⬜ BUG-P2-19 | tcp_comm.py:195 | `'local'` 关键词误匹配回环接口

- **根因**：ethernet 关键词列表包含 `'local'`，会匹配 Linux 的 `lo` 回环接口
- **影响**：回环接口被误判为以太网接口
- **修复**：从 `eth_keywords` 中移除 `'local'`
- **文件**：[src/tcp_comm.py](../src/tcp_comm.py#L195)

### ⬜ BUG-P2-20 | api_server.py:174 | SIGTERM 处理器中 `import uvicorn` 无意义

- **根因**：信号处理器内 `import uvicorn` 既不使用也不影响 uvicorn 行为
- **影响**：无功能影响，但代码有误导性（配合 P0-02 修复时一并处理）
- **修复**：移除这行（配合 P0-02 修复）
- **文件**：[src/api_server.py](../src/api_server.py#L174)

## P3 — 低优先级（7 个）

### ⬜ BUG-P3-21 | build-deb.sh:42 | `rm -rf "$BUILD_DIR"` 无空值保护

- **根因**：如果 `$BUILD_DIR` 变量为空（脚本修改引入 bug），`rm -rf` 危险
- **修复**：脚本顶部添加 `set -u` 或 `rm -rf` 前检查 `[ -n "$BUILD_DIR" ]`
- **文件**：[packaging/linux/build-deb.sh](../packaging/linux/build-deb.sh#L20)

### ⬜ BUG-P3-22 | build-deb.sh:46-54 | 前端构建跳过逻辑可能打包过期版本

- **根因**：`if [ ! -d "$FRONTEND_DIR/dist" ]` 仅在 dist 不存在时构建，源码更新后不会重新构建
- **修复**：移除 if 判断，始终构建，或添加 `--force` 参数
- **文件**：[packaging/linux/build-deb.sh](../packaging/linux/build-deb.sh#L46-L54)

### ⬜ BUG-P3-23 | build-deb.sh | 缺少 `dpkg-deb` 可用性检查

- **根因**：`dpkg-deb` 属于 `dpkg-dev` 包，最小化安装可能没有
- **修复**：脚本开头添加 `command -v dpkg-deb` 检查
- **文件**：[packaging/linux/build-deb.sh](../packaging/linux/build-deb.sh#L20)

### ⬜ BUG-P3-24 | control-cuda:9 | NVIDIA 驱动版本列表不完整

- **根因**：只列了 535/545/550，缺少 470/525/560 等
- **修复**：改为 `nvidia-driver-535 | nvidia-driver-545 | nvidia-driver-550 | nvidia-driver-560`
- **文件**：[packaging/linux/control-cuda](../packaging/linux/control-cuda#L9)

### ⬜ BUG-P3-25 | README:265 | 前置条件缺少 `dpkg-dev`

- **根因**：`dpkg-deb` 来自 `dpkg-dev` 包，未在依赖列表中
- **修复**：在前置条件中添加 `dpkg-dev`
- **文件**：[packaging/README.md](../packaging/README.md#L265)

### ⬜ BUG-P3-26 | README:285 | 缺少 `apt-get install -f` 步骤

- **根因**：`dpkg -i` 后如果依赖不满足，需要 `apt-get install -f` 修复
- **修复**：在安装命令后补充此步骤
- **文件**：[packaging/README.md](../packaging/README.md#L285)

### ⬜ BUG-P3-27 | README:176-183 | 版本号更新清单遗漏 Linux 文件

- **根因**：清单未列出 `build-deb.sh` VERSION、`control-cpu` 和 `control-cuda` Version 字段
- **修复**：添加这三项到版本号更新清单
- **文件**：[packaging/README.md](../packaging/README.md#L176-L183)
