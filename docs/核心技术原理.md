# 二、核心概念&技术原理文档（编码+答辩共用）
> 开发时遇到原理问题直接查阅，同时作为答辩知识储备

## 2.1 算子与算子融合
1. 核心连续算子组合：`LayerNorm → Linear → GELU`
2. 算子定义：深度学习最小原子计算单元，串行执行产生大量显存读写
3. 融合原理：多算子合并为一个执行单元，中间结果保留在GPU寄存器，**不落地全局显存**
4. 项目实现方式：`torch.compile()` 自动融合，无手动CUDA开发
5. 优势：不改变计算逻辑、无精度损失、降低IO、提升推理速度

## 2.2 Q / K / V 与 KV 缓存
1. Q(Query)：当前 Token 专属查询向量，每轮重新计算，**不缓存**
2. K(Key)：历史特征索引标签，一次计算、反复复用
3. V(Value)：历史语义真实数据，与K一一绑定
4. KV缓存作用：自回归解码阶段避免重复计算历史特征，大幅降低算力
5. 传统KV问题：整块连续显存，易产生内存碎片
6. 分页KV（本项目自研简化版）
   - 原理：借鉴操作系统内存分页，拆分固定大小内存页 + 页表映射
   - 功能：动态分配、自动回收、降低显存碎片、提升边缘设备显存利用率
   - 约束：**仅本地节点维护，不跨设备传输**

## 2.3 模型量化
1. 定义：FP32/FP16 浮点权重 → INT8/INT4 整数存储，模型结构不变
2. 类型：后训练量化 PTQ（本项目使用，无需重新训练）
3. 精度取舍：以**微小精度损失**换取显存降低、推理提速
4. 项目配置：权重使用 INT4 量化，中间特征、KV缓存保留 FP16
5. 作用：适配低配边缘笔记本，降低模型静态显存占用

## 2.4 边缘设备 & 分布式流水线推理
1. 边缘设备：本项目指普通学生笔记本，算力/显存有限、网络不稳定
2. 流水线并行规则
   - 单条请求：层间强数据依赖，**严格串行**，无法并行
   - 多条并发请求：不同请求在不同节点同时运行，实现流水线并行
3. 分层策略：根据三台设备性能**自适应分配模型层**，非平均拆分
4. 数据依赖：下一级节点必须接收上一级中间隐藏特征张量才能计算

## 2.5 通信协议（TCP）
1. 底层协议：TCP（可靠传输、丢包重传、保序，适配特征张量传输）
2. 拓扑：主从架构、长连接不断开
3. 数据包格式：`4字节长度头 + 数据体`，解决 TCP 粘包/拆包
4. 传输数据分类
   - 控制数据：指令、心跳、节点状态、文本问答（JSON 字符串）
   - 推理数据：模型中间隐藏特征张量（二进制字节流）
5. 约束：Q/K/V、KV缓存**不跨节点传输**
