# 四、全局配置文件 `config.py`（统一参数，一处修改全局生效）
集中所有硬编码参数，便于调试、切换模式

```python
# 1. 网络配置
SERVER_IP = "192.168.x.x"
SERVER_PORT = 8888
HEARTBEAT_INTERVAL = 3  # 心跳包间隔(秒)

# 2. 模型配置

QUANT_TYPE = "int4"      # fp16 / int8 / int4
USE_COMPILE = True       # 是否开启算子融合

# 3. 分页KV配置
PAGE_SIZE = 128          # 单页容纳Token数量

# 4. 模型分层配置（三台节点起止层）
MAIN_NODE_LAYERS = (0, 12)
CLIENT1_LAYERS = (12, 24)
CLIENT2_LAYERS = (24, 32)

# 5. 运行模式
RUN_MODE = "distributed" # single 单机 / distributed 分布式
```

# 五、运行流程说明（全链路时序）

## 5.1 系统初始化流程
1. 读取全局配置 `config.py`
2. 主节点启动 TCP 服务端，监听端口
3. 两台从节点启动客户端，主动连接主节点，注册节点信息
4. 所有节点加载模型、执行量化、开启算子融合、初始化分页KV缓存
5. 主节点调度器进入就绪状态，等待前端请求

## 5.2 单次推理完整流程
1. 前端接收用户提问，下发至主节点调度器
2. **Prefill 阶段**：主节点处理输入文本，计算前半层 → 生成中间特征 → TCP 发送至从节点1
3. 从节点1接收张量，计算中间层 → 转发特征至从节点2
4. 从节点2计算最后层，生成结果，本轮所有节点**本地追加KV至分页缓存**
5. **Decode 循环阶段**：逐Token生成，重复「计算→传张量→追加KV」流程
6. 生成结束：清空本轮KV缓存，节点回归空闲状态
7. 最终结果回传至前端展示，同步记录所有性能指标

## 5.3 异常处理流程
1. 网络断开：心跳检测失效 → 标记节点离线、尝试自动重连
2. 推理报错：停止任务、清空KV、重置流水线、前端提示错误
3. 显存溢出：日志告警、任务终止
