# 项目总览与整体架构

> 状态：现行
> 更新日期：2026-07-14
> 适用范围：当前已经落地的 PC、Linux 与 Android 产品边界
> 远期架构：[混合分布式推理体系规划](混合分布式推理体系规划.md)

## 1. 项目目标

QLH 面向异构边缘计算设备，不限定为笔记本，也不限定固定节点数量。当前设备范围包括：

- Windows/Linux 台式机、工作站、服务器和笔记本；
- 带 NVIDIA CUDA GPU 的独显节点；
- 仅有 CPU 或集成显卡的 PC 节点；
- Android 手机和平板；
- 后续通过明确能力协议接入的其他边缘设备。

系统解决的是单个设备受内存、显存、算力或能耗限制时的模型运行与协作问题。默认 Qwen-1.8B 只是基础样例，模型注册表还包含其他 Qwen 和 DeepSeek 实验槽位；模型能否参与层流水线必须经过架构、层数、tokenizer、权重和运行时兼容性校验。

## 2. 当前执行路径

| 执行路径 | 节点与引擎 | 模型格式 | 当前定位 |
|----------|------------|----------|----------|
| PC 本地 PyTorch | CUDA PC；必要时可回退 CPU | Safetensors | 完整模型推理 |
| PC PyTorch 层流水线 | 能加载同一 PyTorch 模型工件并通过能力校验的 PC 节点 | Safetensors | 连续 Transformer 层段协作 |
| PC 本地 llama.cpp | CPU/集显 PC | GGUF | 完整模型本地推理 |
| Android 本地 llama.cpp | Android Full | GGUF | 完整模型本地推理 |
| Android 远程模式 | Android Full/Lite -> PC 主节点 | 无本地模型要求 | HTTP 客户端，不参与当前层流水线 |

“PC 集显版”“PC 独显版”“Android Full/Lite”是产品和依赖分级，不直接等价于分布式角色。调度器还必须检查实际引擎、模型身份、模型就绪状态、内存、加速器和网络状态。

## 3. 当前 PyTorch 层流水线

下面只是一个三节点示例，不是固定拓扑：

```text
前端/API
   -> 主节点：Embedding + 首段 Transformer 层
   -> 从节点 A：中间连续层段
   -> 从节点 B：末段 Transformer 层 + LM Head
   -> 主节点汇总结果并返回
```

当前关键约束：

1. 单次执行使用同一款、同一版本且通过身份校验的 PyTorch 模型工件。
2. 主节点参与模型计算，并作为首段 Embedding 锚点至少承担一层；不能仅按设备数量平均分层。
3. 层段必须连续且完整覆盖模型；Embedding 和 LM Head 职责不能落到未加载相应权重的节点。
4. hidden states 在节点间传输，各节点维护自己层段的 KV cache。
5. 节点未完成模型同步、层配置加载和 ready ACK 前不能进入执行路径。
6. 任一阶段失败都必须结束当前 attempt，再按可恢复边界重试或降级，不能把半完成状态继续交给下一节点。

现行消息、就绪与回退细节见[分布式推理流水线实施计划](分布式推理流水线实施计划.md)。

## 4. 软件分层

| 层级 | 主要职责 | 当前实现 |
|------|----------|----------|
| 客户端 | 聊天、设置、节点与执行状态展示 | React、Jetpack Compose |
| API/控制面 | 请求入口、节点管理、模型管理、状态与指标 | FastAPI、`api_server.py` |
| 调度面 | 请求队列、层分配、执行编排、取消、回退 | `scheduler.py`、`graph_orchestrator.py` |
| 通信面 | 注册认证、心跳、控制消息、张量传输 | `tcp_comm.py`、HTTP/Tailscale |
| 推理面 | PyTorch 完整/分层推理、llama.cpp 本地推理 | `model_module.py`、`llama_engine.py` |
| 存储面 | 集群配置、会话、节点状态、离线降级 | PostgreSQL、本地 JSON、Android Room/DataStore |

## 5. 远期边界

现有 `LAYER_FORWARD` 是 PyTorch 层流水线协议，不会扩展成所有设备和模型格式共用的张量协议。完整推理 Worker、任务链、张量并行和 GGUF Stage 将作为独立 Provider 接入统一控制面，详见[混合分布式推理体系规划](混合分布式推理体系规划.md)。
